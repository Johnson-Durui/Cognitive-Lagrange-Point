# 网页版使用说明

现在这个项目已经有一个本地网页控制台，网页里可以：

- 查看实验状态
- 保存 API 配置
- 启动 / 停止实验
- 实时查看 checkpoint 进度
- 刷新星图里的真实发现点

## 启动网页版

在项目根目录执行：

```bash
./run-web.sh
```

然后打开：

```text
http://127.0.0.1:4173
```

## 网页里的三个预设

- `续跑完整版`
  继续当前 checkpoint，打开稳定性、振荡、断层线、隧道效应、社会冲突预测。
- `低成本验证`
  用较短轮次验证链路是否正常，适合先试接口和中转稳定性。
- `从头跑 MVP`
  从阶段一重新开始，只跑最轻的一版。

## 第一次使用

1. 打开“实验控制台”
2. 填入 `Base URL`、`模型名`、`API Key`
3. 点“保存配置”
4. 选择一个预设
5. 点“开始 / 继续”

## 网页看到的内容分别代表什么

- `深筛候选`
  当前进入深度筛选和后续测绘的候选数
- `确认点`
  已通过当前流程确认的认知拉格朗日点数量
- `断层线`
  已识别出的认知断层线数量
- `隧道效应`
  已建立的 CLP 间连接数
- `冲突预测`
  已生成的社会冲突预测条目数

## 和终端版的关系

如果你已经在终端里直接跑了 `research.run`：

- 网页不会重复再起一条实验
- 网页会继续读取 `checkpoint.json`
- 你仍然可以在网页里看到确认点和阶段进度
- 但网页不能直接接管那条终端进程的 stdout

## 结果文件

实验过程中重点会更新这些文件：

- `research/output/checkpoint.json`
- `research/output/results.json`
- `research/output/report.txt`
- `research/output/api_diagnostics.jsonl`
- `discovered-data.js`

跑完后刷新网页，星图会自动读取新的 `discovered-data.js`。
